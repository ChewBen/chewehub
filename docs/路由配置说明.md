# 路由配置说明

## 默认首页变更

已成功将博客风格页面设置为默认首页。

### 路由结构

1. **博客首页（新默认首页）**
   - 路径：`/home`
   - 组件：`@/views/business/forum/index.vue`
   - 特点：独立页面，不使用 Layout（无侧边栏和顶部导航栏）
   - 访问方式：
     - 直接访问：`http://域名/home`
     - 根路径自动重定向：`http://域名/` → `/home`

2. **管理后台首页（保留）**
   - 路径：`/dashboard`
   - 组件：`@/views/index`（原若依管理后台首页）
   - 特点：使用 Layout（有侧边栏和顶部导航栏）
   - 访问方式：`http://域名/dashboard`

### 路由配置位置

配置文件：`cheweHub-ui/src/router/index.js`

### 主要变更

1. 添加了博客首页路由（`/home`），独立页面，不使用 Layout
2. 根路径 `/` 重定向到 `/home`
3. 原管理后台首页移至 `/dashboard` 路径

### 权限控制

当前博客首页**需要登录**才能访问。如果需要设为公开页面，需要：

1. 在 `cheweHub-ui/src/permission.js` 中将 `/home` 加入白名单：

```javascript
const whiteList = ['/login', '/register', '/home']
```

2. 或者修改权限检查逻辑，允许未登录用户访问 `/home`

### 注意事项

1. **登录后重定向**：用户登录成功后，系统默认重定向到 `/`，会自动跳转到博客首页 `/home`
2. **路由顺序**：路由匹配是按照定义顺序进行的，404路由（`/:pathMatch(.*)*`）应该放在最后
3. **Layout组件**：博客首页不使用 Layout，是完整的独立页面布局

### 后续优化建议

1. 如果博客首页需要公开访问，建议加入白名单
2. 可以根据需求调整登录后的默认重定向路径
3. 可以在博客页面添加"进入管理后台"的链接，指向 `/dashboard`
