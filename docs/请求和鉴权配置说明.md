# 请求和鉴权配置说明

## 请求方式

### 基础配置

**请求库：** Axios 1.9.0

**默认配置：**
- Content-Type: `application/json;charset=utf-8`
- 超时时间: 10秒
- BaseURL: 通过环境变量 `VITE_APP_BASE_API` 配置

### 请求示例

```javascript
import request from '@/utils/request'

// GET 请求
request.get('/api/user/list', { 
  params: { page: 1, size: 10 } 
})

// POST 请求
request.post('/api/user/add', { 
  name: 'test', 
  age: 18 
})

// PUT 请求
request.put('/api/user/update', { 
  id: 1, 
  name: 'test' 
})

// DELETE 请求
request.delete('/api/user/1')
```

### 特殊请求配置

**跳过Token验证：**
```javascript
request.get('/api/public/info', {
  headers: { isToken: false }
})
```

**跳过防重复提交：**
```javascript
request.post('/api/user/add', data, {
  headers: { repeatSubmit: false }
})
```

**文件下载：**
```javascript
import { download } from '@/utils/request'

download('/api/file/download', { fileName: 'test.xlsx' }, 'test.xlsx')
```

### 请求拦截器功能

1. **自动添加Token**
   - 从 Cookies 读取 `Admin-Token`
   - 添加到请求头：`Authorization: Bearer {token}`
   - 可通过 `headers: { isToken: false }` 跳过

2. **GET参数处理**
   - 自动将 `params` 转换为 URL 查询字符串
   - 格式：`?key1=value1&key2=value2`

3. **防重复提交**
   - POST/PUT 请求默认启用
   - 1秒内相同请求会被拦截
   - 可通过 `headers: { repeatSubmit: false }` 跳过
   - 使用 sessionStorage 存储请求信息

### 响应拦截器处理

**响应码处理：**
- `200`: 正常返回数据
- `401`: 未授权，弹出确认框，跳转登录页
- `500`: 服务器错误，显示错误消息
- `601`: 业务警告，显示警告消息
- 其他非200: 显示错误通知

**错误处理：**
- Network Error → "后端接口连接异常"
- timeout → "系统接口请求超时"
- HTTP错误码 → "系统接口{code}异常"

## 鉴权方式

### Token存储

**存储方式：** Cookies（使用 `js-cookie`）

**Token键名：** `Admin-Token`

**管理工具：** `src/utils/auth.js`

```javascript
import { getToken, setToken, removeToken } from '@/utils/auth'

// 获取Token
const token = getToken()

// 设置Token（登录成功后）
setToken(token)

// 移除Token（登出时）
removeToken()
```

### Token使用流程

1. **登录**
   ```javascript
   // 登录成功后
   const { token } = loginResponse.data
   setToken(token)
   ```

2. **请求自动携带**
   - 每个请求自动在请求头添加 `Authorization: Bearer {token}`
   - 无需手动添加

3. **Token过期处理**
   - 401响应时自动弹出确认框
   - 用户确认后清除Token并跳转登录页

4. **登出**
   ```javascript
   removeToken()
   // 清除用户信息
   // 跳转登录页
   ```

### 路由守卫

**配置文件：** `src/permission.js`

**白名单路由：**
- `/login` - 登录页
- `/register` - 注册页

**守卫逻辑：**

1. **有Token的情况：**
   - 访问登录页 → 重定向到首页
   - 访问白名单 → 允许访问
   - 访问受保护页面：
     - 无用户信息 → 获取用户信息 → 生成动态路由 → 允许访问
     - 有用户信息 → 直接允许访问

2. **无Token的情况：**
   - 访问白名单 → 允许访问
   - 访问受保护页面 → 重定向到登录页（带redirect参数）

**动态路由：**
- 根据用户角色权限动态生成路由表
- 使用 `router.addRoute()` 添加路由

### 权限控制

**指令方式：**
```vue
<!-- 权限控制 -->
<el-button v-hasPermi="['system:user:add']">添加</el-button>

<!-- 角色控制 -->
<el-button v-hasRole="['admin']">管理员</el-button>
```

**函数方式：**
```javascript
import { useDict } from '@/utils/dict'

// 检查权限
this.$auth.hasPermi(['system:user:add'])

// 检查角色
this.$auth.hasRole(['admin'])
```

## 缓存插件

**文件：** `src/plugins/cache.js`

**使用方式：**
```javascript
import cache from '@/plugins/cache'

// Session存储
cache.session.set('key', 'value')
cache.session.get('key')
cache.session.setJSON('key', { data: 'value' })
cache.session.getJSON('key')
cache.session.remove('key')

// LocalStorage存储
cache.local.set('key', 'value')
cache.local.get('key')
cache.local.setJSON('key', { data: 'value' })
cache.local.getJSON('key')
cache.local.remove('key')
```

**全局使用：**
```javascript
// 在组件中
this.$cache.session.set('key', 'value')
this.$cache.local.get('key')
```

## 常见问题

### Q: Token过期后如何处理？
A: 401响应时会自动弹出确认框，用户确认后清除Token并跳转登录页。

### Q: 如何跳过Token验证？
A: 在请求配置中添加 `headers: { isToken: false }`。

### Q: 如何跳过防重复提交？
A: 在请求配置中添加 `headers: { repeatSubmit: false }`。

### Q: 文件下载如何处理？
A: 使用 `download()` 方法，自动处理blob响应和错误提示。

### Q: 路由守卫如何配置白名单？
A: 在 `src/permission.js` 中修改 `whiteList` 数组。

### Q: 如何实现动态路由？
A: 在路由守卫中调用 `usePermissionStore().generateRoutes()` 生成路由，然后使用 `router.addRoute()` 添加。
